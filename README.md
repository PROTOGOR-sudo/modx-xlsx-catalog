# 🗂️ Каталог продукции из XLSX для MODX Revolution

> **Кейс:** Разработка каталога товаров с фильтрацией, поиском и пагинацией на базе Excel-прайс-листа

![PHP](https://img.shields.io/badge/PHP-5.6%2B-blue?logo=php) ![MODX](https://img.shields.io/badge/CMS-MODX%20Revolution-orange) ![Bootstrap](https://img.shields.io/badge/UI-Bootstrap%204-purple?logo=bootstrap) ![License](https://img.shields.io/badge/license-MIT-green)

---

## 📋 О проекте

Клиент — **Skynet Group** (IT-компания, системный интегратор, B2B).  
Задача: прикрутить к существующему сайту на MODX каталог из Excel-файла с **18 097 позициями** электроники и аксессуаров — без дорогих плагинов и сложных решений.

**Сайт:** [skynet-group.ru](https://skynet-group.ru)

---

## 🎯 Задачи

- Читать прайс-лист `.xlsx` и отображать товары на сайте
- Автоматически определять категории и бренды из названий товаров
- Фильтрация по категории, бренду, наличию
- Полнотекстовый поиск по названию и артикулу
- Два режима отображения: карточки и таблица
- Совместимость с существующей темой сайта (Okler / Bootstrap 4)
- Кнопка скачивания прайс-листа

## 🏗️ Архитектура решения

```
catalog.php                  ← единая точка входа
catalog/
  data/
    catalog.json             ← 18 097 товаров (5.1 MB, генерируется из XLSX)
    catalog.xlsx             ← оригинальный прайс (для скачивания)
  tools/
    xlsx_to_json.php         ← скрипт обновления каталога
  libs/
    SimpleXLSX.php           ← библиотека чтения XLSX (опционально)
```

**Ключевое решение:** вместо чтения XLSX при каждом запросе — однократная конвертация в JSON. Это даёт моментальную загрузку страницы даже при 18 000+ позициях.

## 🔍 Что было сделано

### 1. Анализ данных
Исходный файл имел пустые колонки «Вид оборудования» и «Бренд» — вся информация была только в названии товара.

**Решение:** написан алгоритм автоклассификации на регулярных выражениях:
- **15 категорий**: Кабели, Зарядные устройства, Наушники, Хабы и доки, и др.
- **11 брендов**: UGREEN, JBL, Satechi, Belkin, Xiaomi, Samsung, и др.
- **3 статуса наличия**: В наличии (с количеством), В резерве, Нет

### 2. Конвертация данных
Python-скрипт с `openpyxl` конвертирует XLSX в оптимизированный JSON:
```
18 097 товаров → 5.1 MB JSON → сжатие до 0.7 MB (86% gzip)
```

### 3. Разработка каталога
- Фильтрация на стороне сервера (PHP array_filter)
- Пагинация: 48 товаров на страницу
- URL-параметры для всех фильтров (совместимо с историей браузера)
- Счётчики товаров в каждой категории/бренде обновляются с учётом активных фильтров

### 4. UI/UX
- Sticky-шапка и sticky-сайдбар
- Toggle-переключатель «Только в наличии»
- Цветные бейджи статуса (зелёный/жёлтый/красный)
- Переключение карточки ↔ таблица с сохранением в localStorage
- Активные фильтры отображаются как «чипы» с кнопкой удаления
- Адаптивная сетка: 3 → 2 → 1 колонка

### 5. Интеграция с MODX
- Файл работает независимо от MODX (не требует плагинов)
- Подключает CSS/JS темы сайта напрямую
- В меню MODX добавлен Weblink-ресурс «Каталог» → `/catalog.php`

## 🚀 Установка

### Требования
- PHP 5.6+
- Права на файлы: `644` для файлов, `755` для папок

### Шаги

1. **Скопировать файлы** в корень сайта:
```bash
catalog.php → /var/www/your-site/catalog.php
catalog/    → /var/www/your-site/catalog/
```

2. **Выставить права:**
```bash
chmod 644 catalog.php
chmod -R 755 catalog/
chmod 644 catalog/data/catalog.json
chmod 644 catalog/data/catalog.xlsx
```

3. **Открыть** `ваш-сайт.ru/catalog.php`

### Обновление прайса

При получении нового XLSX:
1. Заменить `catalog/data/catalog.xlsx`
2. Скачать [SimpleXLSX.php](https://github.com/shuchkin/simplexlsx/raw/master/src/SimpleXLSX.php) → `catalog/libs/`
3. Запустить: `php catalog/tools/xlsx_to_json.php`

## 💡 Особенности реализации

**Без внешних зависимостей на продакшене** — каталог читает готовый JSON, никаких библиотек не требуется для работы сайта.
